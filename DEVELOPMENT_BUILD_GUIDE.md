## ðŸ› Troubleshooting

### Common Issues

**"No development build found" error:**

- Make sure you've installed the development build first
- Verify the bundle identifier matches your app config
- Try rebuilding: `eas build --profile development --platform [ios/android]`

**Metro bundler connection issues:**

- Ensure your device/simulator is on the same network
- Try restarting Metro: `yarn dev`
- Clear Metro cache: `yarn start --reset-cache`

**Build failures:**

- Check the build logs on expo.dev
- Verify all dependencies are compatible
- Run `npx expo install --check` to fix version conflicts

## Troubleshooting

### Common issues:

1. **Build fails due to dependencies**: Run `npx expo install --check` to fix version mismatches
2. **Simulator not found**: Make sure iOS Simulator is open before installing
3. **App doesn't connect**: Make sure both the development server and app are running

### Dependencies to fix (as noted by expo doctor):

- Remove `package-lock.json` (you're using yarn)
- Update `expo-auth-session` to `~6.0.3`
- Update `expo-crypto` to `~14.0.2`
- Add `.expo/` to `.gitignore`

You can fix these later with:

```bash
rm package-lock.json
npx expo install --check
echo ".expo/" >> .gitignore
```
